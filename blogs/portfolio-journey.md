# My Portfolio Journey: From VS Code to Mobile Constraints and the Next Step with React

**Published:** May 15, 2023
**Author:** Abel Arroyo
**Tags:** Web Development, Portfolio, React, Mobile Development

## Introduction

Building a personal portfolio website is an essential step for any developer. This post chronicles my journey from initial development in VS Code through tackling mobile responsiveness challenges to my current transition toward building a more dynamic version with React.

## The Beginning with VS Code

My portfolio journey began with VS Code, HTML, CSS, and JavaScript. The initial development process was straightforward but presented its own learning opportunities:

- Setting up a proper project structure
- Implementing modern CSS techniques
- Creating responsive layouts
- Adding interactive elements with vanilla JavaScript

## Mobile Constraints and Challenges

As I moved to test the site on various devices, I encountered several challenges with mobile responsiveness:

1. **Viewport inconsistencies** across different mobile devices
2. **Touch interactions** that needed special handling compared to mouse events
3. **Performance optimization** for slower mobile connections
4. **Layout shifts** that required careful CSS adjustments

The process of adapting my portfolio for mobile taught me valuable lessons about progressive enhancement and the importance of testing on real devices.

## The Next Step: React Implementation

After gaining more experience with React, I decided to rebuild my portfolio with this framework to leverage:

- **Component-based architecture** for better code organization
- **State management** for more dynamic user experiences
- **React routing** for smoother page transitions
- **Better maintainability** for future updates

## Conclusion

My portfolio continues to evolve as I grow as a developer. What started as a simple HTML/CSS project has become a playground for implementing new techniques and showcasing my growing skill set.

The journey from basic web development to responsive design to React implementation mirrors my own growth path as a developer, and I'm excited to see where this journey takes me next.
